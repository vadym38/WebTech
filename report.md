# Звіт до лабораторної роботи

## Тема

Створення основи SPA-додатку на ReactJS для сервісу пошуку репетиторів.

## Мета роботи

Розробити основу односторінкового вебдодатку, реалізувати сторінки відповідно до індивідуального завдання, навігацію між ними, авторизацію, компонент із власним станом, відображення даних з API, форму редагування та управління станом додатку.

## Індивідуальне завдання

Тема проєкту: сервіс пошуку репетиторів.

Основні сутності:

- викладач;
- предмет;
- учень;
- урок;
- розклад;
- відгук;
- оплата.

## Використані технології

- HTML5;
- CSS3;
- JavaScript;
- ReactJS;
- PHP;
- MySQL;
- AJAX `fetch`;
- JSON;
- Docker;
- Render.

## Структура проєкту

- `index.html` - головна HTML-сторінка SPA-додатку;
- `style.css` - стилі інтерфейсу;
- `app.js` - ReactJS-логіка додатку;
- `index.php` - серверне API для роботи з базою даних;
- `Dockerfile` - інструкції для запуску додатку на хостингу;
- `render.yaml` - конфігурація сервісу Render;
- `start_spa.bat` - запуск локального PHP-сервера;
- `README.md` - коротка інструкція запуску.

## Реалізовані сторінки SPA

У додатку реалізовано такі сторінки:

- `Головна` - короткий опис системи та статистика;
- `Уроки` - таблиця уроків із фільтрацією, сортуванням і підстановкою даних зі зв'язаних таблиць;
- `Редагування` - форма зміни даних обраного уроку;
- `Авторизація` - форма входу користувача.

Навігація між сторінками відбувається без перезавантаження документа. Поточна сторінка зберігається у стані React-додатку.

## Авторизація

У лабораторній роботі реалізовано навчальну авторизацію. Користувач вводить логін і пароль, після чого дані користувача зберігаються в `localStorage`. Після авторизації стає доступним редагування даних.

## Компонент із власним станом

Компонент `LocalCounter` має власний локальний стан, створений за допомогою `React.useState`. Він містить кнопки збільшення, зменшення та скидання значення. Події кнопок змінюють стан компонента без перезавантаження сторінки.

## Відображення даних

Дані завантажуються з API за допомогою AJAX-запитів `fetch`. Формат обміну даними між клієнтом і сервером - JSON.

Додаток отримує дані з таблиць:

- `lesson`;
- `teacher`;
- `subject`;
- `student`;
- `schedule`;
- `review`;
- `payment`.

Основною таблицею є `lesson`. Замість кодів зв'язаних таблиць у таблиці відображаються зрозумілі дані: ім'я викладача, назва предмета, ім'я учня, розклад, відгук та оплата.

## Фільтрування та сортування

На сторінці `Уроки` реалізовано:

- фільтр за викладачем;
- фільтр за предметом;
- фільтр за учнем;
- текстовий пошук;
- сортування за номером, викладачем, предметом, учнем або розкладом.

## Форми та редагування даних

На сторінці `Редагування` створено форму для зміни даних уроку. Після натискання кнопки `Зберегти` додаток надсилає `PUT`-запит на сервер. Сервер оновлює запис у базі даних і повертає оновлений об'єкт у форматі JSON.

## Управління станом додатку

Глобальний стан реалізовано за допомогою `React.useReducer`. У стані зберігаються:

- поточна сторінка;
- авторизований користувач;
- списки даних з API;
- обраний урок;
- повідомлення про помилки та успішні дії.

Такий підхід є аналогом централізованого управління станом і може бути розширений у наступних лабораторних роботах.

## Підготовка до хостингу

Для публікації проєкту підготовлено Docker-конфігурацію. Це дозволяє розгорнути додаток на Render як web service. Сервер запускає PHP-додаток на порту, який передає хостинг через змінну середовища `PORT`.

API підтримує два режими роботи:

- локальний режим з MySQL-базою `tutors`;
- демо-режим із JSON-сховищем, якщо MySQL недоступна на хостингу.

Завдяки цьому сайт можна запустити локально через XAMPP або опублікувати на Render без додаткового налаштування бази даних.

## Висновок

У ході лабораторної роботи було створено основу SPA-додатку на ReactJS для сервісу пошуку репетиторів. Реалізовано навігацію між сторінками, авторизацію, компонент із власним станом, завантаження та відображення даних з API, фільтрування, сортування, форму редагування, централізоване управління станом додатку та підготовку до розгортання на Render.
